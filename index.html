When a user makes a request, Oryxa translates their words into a call to a function you‚Äôve defined in your app, then hands control back to your app‚Äôs logic. Oryxa is built for performance and reliability, with zero reflection or runtime scanning. Your app logic stays secure, fast, and easy to maintain.

Oryxa is designed to fit right into modern app architectures. To keep integration simple and scalable, and relies on Hilt for dependency injection. This ensures that your app‚Äôs data, actions, and AI-powered features are all wired together cleanly and automatically.

What you need to get started:

  • You‚Äôll define your app‚Äôs actions as simple functions, using primitive parameters (text, numbers, booleans...).
  • For dynamic lists and lookups, just provide a simple ‚Äúlookup provider‚Äù to tell Oryxa about your available values(e.g., task types, categories, etc).

With these in place, Oryxa auto-generates all the boilerplate for connecting user requests to your app‚Äôs logic, so you can focus on building great features.

How it works!

Initialize OryxaAIHost using your ViewModel and application context. This object bridges your annotated logic with the AI interface. 

// Inside your Activity or Fragment
val host = OryxaAIHost(
    viewModel = viewModel,
    context = requireContext()
)

Detect supported features

The generated metadata from @ExposeToAI allows you to query accessible functions. Useful for debugging or dynamic chat suggestions. 


val features = host.getAvailableFunctions()
features.forEach {
    println("AI Callable: ${it.name}")
}

Add the dependency

Add Oryxa to your build.gradle.kts file:


dependencies {
    implementation("ai.oryxa:oryxa-core:1.0.0")
    ksp("ai.oryxa:oryxa-processor:1.0.0")
}

Enable the KSP plugin:


plugins {
    id("com.google.devtools.ksp") version "1.9.0-1.0.13"
}
